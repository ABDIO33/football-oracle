الموضوع: مشروع توقع نتائج كرة القدم — الوضع الحالي والخطة للوصول للقمة

---

نحن نبني نظام توقع exact score لكرة القدم. عندنا حاليًا:

1. 100,540 مباراة في قاعدة البيانات
2. مباشر + walkforward (Elo, form, xG)
3. XGBoost Direct Score Model — 71 feature, 13.74% exact, 49.57% 1X2, RPS 0.126
4. نموذج FD (odds only) — 10.42% exact
5. وحدة كأس العالم 2026 — 96 مباراة متبقية مع venue/ارتفاع
6. سكريب SofaScore شغال (curl_cffi + rate limit 0.5s)
7. open-meteo.com للطقس (مجاني، بدون مفتاح)
8. سكريب FotMob موجود (HTML scraping)

المشكلة: وصلنا لسقف ~13.7% ولا نقدر نتخطاه بالمصادر الحالية.

اللي جرّبناه وفشل:
1. زيادة الميزات الهندسية (24 ميزة جديدة) → 13.74% (فرق 0.08%)
2. LightGBM بدل XGBoost → 12.08% (أسوأ)
3. Hyperparameter tuning → 13.54% (نفس الشيء)

اللي اكتشفناه من فحص المصادر:
1. SofaScore عنده referee (الاسم والبلد) في /event/{id} — ما كنا نستخدمه
2. SofaScore عنده venue (الملعب + المدينة) — ما كنا نستخدمه
3. SofaScore عنده hasEventPlayerStatistics: True — إحصائيات لكل لاعب
4. open-meteo.com مجاني ويعطي طقس تاريخي من 1940 بدون rate limit
5. FotMob ما عنده API صافي — HTML scraping فقط، أبطأ وأهش من SofaScore
6. API-Football المفتاح ميت (403)
7. RSSSF أرشيف تاريخي فقط — لا يفيد في التنبؤ

الفجوة الحقيقية للوصول للأفضل عالمياً (25% exact):
- المعلومة الوحيدة الناقصة: إصابات وغيابات اللاعبين قبل المباراة
- 70% من الفرق بين 13.7% و25% هو معرفة "من لا يلعب الليلة"
- SofaScore عنده إحصائيات لاعبين لكل مباراة → نقدر نبني Player Model بأنفسنا
- نجمع player stats لكل المباريات → نتعلم: "لما اللاعب X يغيب، الفريق يضعف قد إيه؟"

الخطة:
1. STADIUM_DB — نبني قاعدة بيانات 200 ملعب بإحداثياتهم
2. open-meteo — نضيف طقس لكل مباراة (قديم + جديد)
3. Referee — نستخرج من SofaScore ونضيفه كـ feature
4. Player Model — نبني تتبع لكل لاعب (إصاباتنا بأنفسنا من SofaScore)
5. Continuous Learning — كل خطأ نحلله ونضيف feature

بعد هذي الخطوات: نتوقع 20-25% exact. بدونها: 13.7% سقف.
